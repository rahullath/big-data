0. Current State Audit

What exists and works

  • Model B preprocessing (preprocessing_b.py) — fetches 13 market indices + VIX, YIELD_SPREAD, TED_SPREAD, CREDIT_SPREAD via yfinance + FRED
  • Feature engineering (feature_engineering_b.py) — rolling Pearson correlation (60d), permutation entropy, rolling volatility, eigenvalue ratio, volatility synchrony, all 78 pairwise correlations
  • Model training (models_b.py) — Random Forest with walk-forward validation, SHAP via TreeExplainer, crisis prediction validation (2008, 2020)
  • Regime labelling (regime_labeling_b.py) — HEDGE / SPECULATIVE / PONZI
  • Target variable is ISE_USD — USD-denominated. Good. Do not change.
  • Export to JSON exists (export_json_b.py)

Critical problems

  • Walk-forward splits only test 2008 and 2020. The 2018 TRY crisis and the 2021–26 ongoing Turkey crisis are never validated. This is the core academic finding the dashboard is built around. Both must be added.
  • No OLS or VAR models implemented. models.py and train_pipeline.py are empty files. Dashboard comparison table needs all three models.
  • Export JSON format from export_json_b.py does not match the schema the dashboard expects (fragility_output.json with models.model_2009 and models.model_2003 keys). Need a new unified exporter.
  • Model A (2009–2011 original) pipeline is effectively empty — preprocessing.py, models.py, train_pipeline.py are all empty. Model A exists only as simulated data in the dashboard right now.
  • No DXY (USD Index) in feature set. Research confirms this is the single most important external driver of TRY weakness. Missing.
  • No Brent Crude, EUR/USD, US 10Y Yield in feature set. All are in the research recommendations and free to fetch.
  • No CBRT governor turnover dummy. Four events (Mar 2019, Jul 2019, Nov 2020, Mar 2021) each caused immediate lira spikes. Zero-cost, high-signal feature.
  • Monthly aggregation is missing. Dashboard operates on monthly scores. Pipeline outputs daily data. Need a monthly resampling step before JSON export.
  • Fragility score formula in feature_engineering_b.py has 7 components with dynamic weight redistribution when macro signals are missing. Model A (to be rebuilt) must use a simpler fixed 4-component formula so the two models are genuinely comparable.

What exists in the original dataset (Group_5.csv)

  • 8 indices: SP500, DAX, FTSE, NIKKEI, BOVESPA, EU, EM, ISE (2009-01 to 2011-07)
  • This is Model A's training data. It is complete and sufficient.

fragility_score_b = (
    0.25 * corr_norm           +  # mean rolling correlation (60d)
    0.20 * pe_inv_norm         +  # permutation entropy, inverted

5. Monthly Resampling

Dashboard operates on monthly granularity. Add this step at end of both pipelines, before JSON export:

def resample_to_monthly(df: pd.DataFrame) -> pd.DataFrame:
    """
    Resample daily data to monthly (last trading day of each month).
    """
    # For prices/levels: take last observation of each month
    # For returns: sum (or mean, depending on convention — use mean for consistency)
    # For scores: take last observation (preserves end-of-month reading)
    
    monthly = df.resample('ME').agg({
        'fragility_score': 'last',
        'regime': 'last',
        'mean_corr': 'mean',
        'permutation_entropy': 'mean',
        'rolling_volatility': 'mean',
        'eigenvalue_ratio': 'mean',
        # Market returns: mean over month
        'SP500': 'mean', 'DAX': 'mean', 'FTSE': 'mean',
        # etc for all features
    })
    
    # Correlations: take end-of-month snapshot (last value)
    corr_cols = [c for c in df.columns if '_' in c and any(
        idx in c for idx in ['SP500','DAX','FTSE','NIKKEI','BOVESPA','EU','EM']
    )]
    for col in corr_cols:
        monthly[col] = df[col].resample('ME').last()
    
    return monthly.dropna(subset=['fragility_score'])

9. What NOT to Add

  • Turkey CDS 5Y spreads — high signal but requires paid data (Bloomberg/Refinitiv). Skip unless your institution has access. CREDIT_SPREAD from FRED is a reasonable proxy.
  • Google Trends "Turkey lira" — interesting but hard to backtest cleanly and may not be accepted academically. Skip for assignment.
  • LSTM / neural networks — LO1 asks for DL but the spec notes "not enough observations to justify." If you include it, use it as a comparison (not the primary model) trained on the extended dataset only. Overfitting risk is real with ~280 monthly observations.
  • EM carry trade index — composite, expensive, not free. DXY + VIX already captures the same signal.
  • Real interest rate (policy rate minus CPI) — high signal but requires CBRT rate data + TurkStat CPI, both patchily available pre-2010. Use US_10Y_YIELD as a proxy for global rate pressure instead.
