Pipeline JSON Schema

Financial Fragility Clock · Group 5 · MSc FinTech Birmingham · v2.0

How to use this document: Your Python pipeline (export_json.py) must produce a single file matching this schema. The dashboard reads it at build time as a static import. Run python export_json.py after every model re-train to regenerate the file.

Output file

A single file: src/data/fragility_output.json

The dashboard imports it directly. No backend or API required — Vite bundles it at build time.

Model object schema

Both model_2009 and model_2003 share identical structure. Repeat for each model.

meta required

{
  "meta": {
    "id":               "model_2009",
    "label":            "2009–2011 Original",
    "training_start":   "2009-01-01",         // ISO date
    "training_end":     "2011-12-31",
    "scoring_start":    "2003-01-31",         // full scored range
    "scoring_end":      "2026-04-21",
    "n_features":       7,
    "feature_names":    ["sp500_ret", "dax_ret", "ftse_ret",
                          "nikkei_ret", "bovespa_ret", "eu_ret", "em_ret"],
    "target":           "ise_usd_ret"          // MUST be USD-denominated!
  }
}
Critical: Use ise_usd_ret (USD-denominated ISE returns), NOT raw TRY-denominated BIST100 levels. In TRY terms the index rose 479% 2010–22 while the dollar value collapsed. Using TRY levels means your model is partly predicting inflation, not fragility.

monthly_scores required

One record per month end. This drives the clock, timeline chart, and scrubber.

{
  "monthly_scores": [
    {
      "date":            "2009-01-31",   // ISO date, end of month
      "fragility_score": 45.2,           // 0–100, your composite score
      "regime":          "SPECULATIVE",  // "HEDGE" | "SPECULATIVE" | "PONZI"
      "components": {
        // The four weighted sub-scores before combining
        "rolling_correlation_60d": 0.72,  // 0–1, weight 0.40
        "permutation_entropy_inv":  0.34,  // 0–1, weight 0.30
        "rolling_volatility_30d":   0.65,  // 0–1, weight 0.20
        "rf_prediction_error":      0.21   // 0–1, weight 0.10
      },
      "correlations": {
        // 60-day rolling Pearson vs ISE_USD for each index
        "sp500":   0.71,
        "dax":     0.65,
        "ftse":    0.58,
        "nikkei":  0.42,
        "bovespa": 0.53,
        "eu":      0.61,
        "em":      0.68
      },
      "features": {
        // Raw normalised feature values for this month
        // (used for lead time analysis charts)
        "sp500_ret":   -0.012,
        "dax_ret":     0.008
        // ... all features
      }
    }
    // ... one object per month, chronological
  ]
}

Regime labelling logic

Apply consistently in both Python and the dashboard:

# Python (feature_engineering.py)
def label_regime(score):
    if score >= 70: return "PONZI"
    elif score >= 40: return "SPECULATIVE"
    else: return "HEDGE"

# Fragility score formula (normalise each component 0-1 first)
fragility_score = (
    0.40 * rolling_correlation_60d +
    0.30 * permutation_entropy_inv +
    0.20 * rolling_volatility_30d_norm +
    0.10 * rf_prediction_error_rolling
) * 100

Python export function

# export_json.py — minimal structure
import json
from datetime import datetime

def export_model(model_id, meta, performance, shap, monthly_scores):
    return {
        "meta": meta,
        "performance": performance,
        "shap_values": shap,
        "monthly_scores": monthly_scores
    }

output = {
    "version": "2.0",
    "generated_at": datetime.utcnow().isoformat() + "Z",
    "models": {
        "model_2009": export_model("model_2009", meta_a, perf_a, shap_a, scores_a),
        "model_2003": export_model("model_2003", meta_b, perf_b, shap_b, scores_b),
    }
}

with open("src/data/fragility_output.json", "w") as f:
    json.dump(output, f, indent=2, default=str)

print(f"Exported {len(output['models'])} models, "
      f"{len(scores_b)} monthly observations")
Validation tip: After generating the JSON, check monthly_scores length matches expected months. For 2003–2026 that's ~280 records. If you have gaps (missing months), forward-fill before export — the dashboard expects a continuous series with no missing dates.

Wiring the dashboard to real data

Replace the two simulation functions in the dashboard JS with real data reads. Swap these functions:

// REPLACE THIS (mock simulation):
function monthToScoreB(idx) { ... }
function heatmapCorr(idx) { ... }

// WITH THIS (real data):
import fragData from './data/fragility_output.json'

const MODEL_B_SCORES = fragData.models.model_2003.monthly_scores;
const MODEL_A_SCORES = fragData.models.model_2009.monthly_scores;

function monthToScoreB(idx) {
  return MODEL_B_SCORES[idx]?.fragility_score ?? 0;
}
function heatmapCorr(idx) {
  const corrs = MODEL_B_SCORES[idx]?.correlations ?? {};
  return ['sp500', 'dax', 'ftse', 'nikkei', 'bovespa', 'eu', 'em']
    .map(k => corrs[k] ?? 0);
}
